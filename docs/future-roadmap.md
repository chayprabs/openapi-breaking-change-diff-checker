# Future Roadmap

This document lists **non-blocking** follow-up work after the launch of OpenAPI Diff.

## Product Expansion

- GraphQL schema compatibility checks
- JSON Schema compatibility checks
- Protobuf and Avro contract review workflows
- Release and CI drift tooling
- Database migration and rollback safety tooling

## Account Features

- saved reports
- team-level ignore rules
- private sharing
- history and audit trails
- workspace presets per repo or service

## Collaboration

- commentable shared reports
- approval or sign-off states
- richer PR review integrations
- organization-wide compatibility policies

## CI And Automation

- first-party CLI parity with the browser engine
- downloadable config files for CI systems
- GitHub App or GitLab integration
- recurring repo checks for published specs

## Engine Depth

- broader schema keyword support
- deeper remote-ref strategies with strict privacy controls
- additional risk heuristics for SDK generation and mobile release cadence
- improved explanations for inconclusive schema features

## Reporting

- CSV export if real user demand appears
- report theming and print tuning
- side-by-side finding diffs for changed schema branches
- richer migration-note generation

## Operational Hardening

- formal rate limiter backed by shared infrastructure if backend traffic grows
- production feedback endpoint with storage and triage workflow
- optional analytics dashboards once a provider is selected
- richer synthetic monitoring for the safe fetch proxy route

## Design And Content

- product brand pass once more tools are live
- dedicated docs site if repo docs stop being sufficient
- tool comparison pages for adjacent compatibility workflows
- more launch content, demos, and case studies
